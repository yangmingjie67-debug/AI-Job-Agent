RAG 是 Retrieval-Augmented Generation 的缩写，中文通常称为检索增强生成。它的基本思路是先从知识库中检索与问题最相关的文本片段，再把这些片段作为上下文交给语言模型。与只依赖模型参数的回答方式相比，RAG 可以引用项目文档、岗位资料和企业知识，降低回答脱离事实的风险。本阶段的 AI Job Agent 只实现 RAG 的基础检索骨架，暂时不让大语言模型根据检索结果生成最终答案。

在一个基础 RAG 流程中，文档首先被读取和清洗，然后被切分成长度适中的文本块。每个文本块会保留来源文件、页码和文件类型等 metadata，并通过 Embedding 模型转换为向量。查询问题也使用同一个 Embedding 模型转换为向量，向量数据库根据距离计算相似度，返回最相关的若干片段。相同模型、稳定的 chunk_id 和持久化向量库可以让重复入库更加可靠。

FastAPI 是一个基于 Python 的现代 Web 框架，适合构建 API 服务。它支持类型提示、自动生成接口文档和异步请求处理。在 AI 应用中，FastAPI 可以作为前端、任务队列和模型服务之间的接口层，例如提供文档上传、检索、问答和健康检查接口。本项目当前使用 Flask 承载网页应用，后续如果需要拆分成前后端服务，可以考虑使用 FastAPI 构建独立的 RAG API，但本阶段不会改变现有 Flask 业务。

Chroma 是一个面向 AI 应用的向量数据库，可以保存文本、Embedding 和 metadata，并提供相似度检索能力。本项目使用 Chroma PersistentClient，把数据持久化到 knowledge_base/chroma_db/，集合名称为 job_agent_knowledge。入库时使用稳定的 chunk_id 和 upsert，重复运行入库脚本不会因为相同文本而无限增加重复记录。检索结果会返回原文、来源、页码、distance 和换算后的 score，便于命令行检查。

AI Job Agent 是一个面向求职场景的 Python 与 Flask 应用。现有功能包括 AI 智能聊天、PDF 简历解析、岗位 JD 分析、简历与岗位匹配、综合匹配度展示、优势和短板分析、简历建议、面试建议、打招呼语生成以及分析历史记录管理。原有功能通过 DeepSeek API 提供模型能力，数据库使用 SQLite 保存用户和分析记录，前端使用 Flask 模板。RAG 模块作为独立基础设施，不会修改这些已有功能。

简历和岗位 JD 的匹配流程通常包括几个步骤。第一步，用户上传 PDF 简历，系统提取其中的项目经历、技术栈和个人信息。第二步，用户输入完整的岗位 JD，系统识别岗位职责、必备技能和加分项。第三步，系统将简历文本和岗位描述进行对照，分析已经匹配的技能、缺少的能力以及需要补充的项目表达。第四步，系统生成匹配度、简历修改建议和面试准备建议。未来接入 RAG 后，可以把项目文档、技术说明和求职知识作为检索上下文，帮助用户获得更贴合事实的准备建议。

本示例文档用于验证中文和英文混合文本的切块、Embedding 和相似度检索。测试问题可以使用“AI Job Agent 如何匹配简历和岗位 JD？”，预期结果应包含简历提取、岗位要求分析、技能差距、匹配度和针对性建议等内容。通过再次执行入库脚本，可以观察 Chroma 集合记录数保持稳定，验证稳定 ID 与 upsert 的去重效果。
